The idea

A small box on the office table. Press the button → it records everyone from a USB mic. Press again → it stops and uploads the audio to Fireflies for transcription, all on its own. A little screen shows what it's doing. Power it on and it just works — nothing depends on anyone's laptop.

How it works

1
Press to record. The button is a plain USB device; the Pi reads the press and starts capturing the mic with ffmpeg.
2
Press to stop. The recording is saved and the device is immediately ready again — uploading happens in the background, never blocking the next chat.
3
Stored safely. The audio is uploaded to a private S3 bucket (kept 30 days) — durable the instant it lands, so nothing is ever lost.
4
Sent to Fireflies. Fireflies can't take a file directly — it takes a link. The device hands it a temporary signed link to the S3 file, and Fireflies fetches & transcribes it.
5
Confirmed. The device quietly checks back that the transcript actually appeared. If one ever goes missing, the screen says so — it never fails silently.

Key design decisions

S3 is the safety net

Audio is durable in cloud storage the moment it's recorded. A transcription hiccup can never lose a recording or jam the device.

It tells you if it failed

The device polls Fireflies to confirm each transcript appeared, and shows a warning if one didn't — no silent losses.

Screen-only feedback

One clear status screen. Audio confirmation was dropped — it was redundant and fragile to get working.

Survives WiFi drops & reboots

Uploads queue and retry across network blips and power cycles. It waits for the clock to sync before uploading.

Locked-down access

A narrowly-scoped cloud key (this device's folder only), private bucket, secrets locked to the device.

Self-starting

Runs as a boot service — power on and it's ready, no laptop, no manual steps, restarts itself if it crashes.

Building it — the shape

1
Flash the SD card with Raspberry Pi OS (with WiFi & SSH preset), plus the display settings.
2
Plug it together and boot — mic, button, monitor, power.
3
Set up cloud bits — an S3 bucket + a scoped key, and a paid Fireflies account key.
4
Install the software and enable it to run on boot.
5
Test every path — button, recording, upload, transcript, screen — end to end.

Before you build — two things to confirm

⚠️
Fireflies must be a paid plan. The free plan can't upload audio via the API at all — so the device needs a paid (ideally dedicated) Fireflies account.
⚠️
The 7" screen is the fiddly part. Getting a cheap HDMI panel to display reliably on a headless Pi is the one real bring-up risk — the exact screen settings are tuned once during setup.
